🛡️ 5권: 보안 편

DB & 서버 보안

    보안이 왜 중요한가

    실제 사례: 의료 데이터 유출

    2023년, 국내 한 병원에서 환자 정보 10만 건이 유출되었습니다.

    • 이름, 주민번호, 연락처
    • 진료 기록, 처방 내역
    • 보험 정보

    결과:

    • 과징금 수억 원
    • 집단 소송
    • 병원 신뢰도 추락
    • 환자 이탈

    의료 데이터가 특별한 이유

    구분 일반 데이터 의료 데이터
    민감도 중간 매우 높음
    법적 규제 개인정보보호법 개인정보보호법 + 의료법
    유출 시 피해 스팸, 사기 차별, 협박, 사회적 낙인
    처벌 수준 과태료 형사처벌 가능

    힐링손 시스템의 민감 데이터

    patients 테이블
├── name (이름)
├── phone (연락처)
├── birth_date (생년월일)
├── address (주소)
└── insurance_code (보험정보)

treatment_sessions 테이블
├── 치료 내용
├── 진단 정보
└── 담당 치료사

billings 테이블
├── 치료비
├── 결제 정보
└── 보험 청구 내역

    이 모든 정보가 유출되면 안 됩니다!

    공격자는 누구인가

    1. 외부 해커

    • 금전 목적 (랜섬웨어, 데이터 판매)
    • 자동화된 공격 도구 사용
    • 취약한 시스템을 무작위로 스캔

    2. 내부 위협

    • 불만을 품은 직원
    • 실수로 인한 유출
    • 과도한 권한 부여

    3. 경쟁자

    • 환자 정보 탈취
    • 영업 비밀 유출

    주요 공격 유형

    공격 대상 피해
    SQL 인젝션 데이터베이스 데이터 탈취/삭제
    XSS 사용자 브라우저 세션 탈취, 피싱
    CSRF 사용자 권한 무단 작업 실행
    무차별 대입 로그인 계정 탈취
    중간자 공격 네트워크 통신 도청

    보안의 3요소: CIA

    C - Confidentiality (기밀성)

    허가된 사람만 데이터를 볼 수 있어야 함

    • 암호화
    • 접근 제어
    • 인증

    I - Integrity (무결성)

    데이터가 무단으로 변경되면 안 됨

    • 입력 검증
    • 해시 검증
    • 로깅

    A - Availability (가용성)

    필요할 때 데이터에 접근할 수 있어야 함

    • 백업
    • 이중화
    • DDoS 방어

    보안 vs 편의성

    보안 강화
├── 장점: 안전
└── 단점: 불편

예시:
├── 복잡한 비밀번호 정책 → 사용자 불편
├── 2단계 인증 → 로그인 시간 증가
├── IP 제한 → 외부 접속 불가
└── 짧은 세션 만료 → 자주 재로그인
    균형이 중요합니다.
    너무 불편하면 사용자가 우회 방법을 찾고, 그게 더 위험합니다.

    법적 요구사항

    개인정보보호법

    • 개인정보 수집 시 동의 필요
    • 안전성 확보 조치 의무
    • 유출 시 통지 의무

    의료법

    • 진료기록 보존 의무 (10년)
    • 의료정보 누설 금지
    • 전자의무기록 보안 기준
    위반 시 처벌
    - 5년 이하 징역 또는 5천만원 이하 벌금
    - 과징금 최대 매출의 4%
    - 영업정지

    이번 장 정리

    • 의료 데이터는 특별히 민감합니다
    • 공격자는 외부 해커, 내부 직원, 경쟁자 등 다양합니다
    • 보안의 3요소: 기밀성, 무결성, 가용성
    • 법적 요구사항을 반드시 지켜야 합니다
    다음 장에서는 가장 흔하고 위험한 공격인 SQL 인젝션을 배웁니다!
