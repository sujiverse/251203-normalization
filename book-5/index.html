    5권: DB & 서버 보안

    환자 정보를 지켜라!

    4권까지 열심히 만든 시스템... 해킹당하면 어떻게 될까요?

    • 환자 개인정보 유출
    • 진료 기록 삭제/변조
    • 매출 데이터 조작
    • 법적 책임 + 신뢰 상실
    의료 데이터는 특히 민감합니다.
    개인정보보호법, 의료법 위반 시 강력한 처벌을 받습니다.

    이 책에서 배우는 것

    Part 1: 웹 공격 이해와 방어

    • SQL 인젝션 - DB를 직접 공격하는 기법
    • XSS - 사용자 브라우저를 공격
    • CSRF - 사용자 권한을 도용

    Part 2: 인증과 권한

    • 안전한 로그인 - 세션, 토큰 관리
    • 비밀번호 저장 - 해싱, 솔팅
    • DB 권한 관리 - 최소 권한 원칙

    Part 3: 데이터 보호

    • 암호화 - 민감 데이터 보호
    • 접근 제어 - 누가 뭘 볼 수 있나
    • HTTPS - 통신 구간 암호화

    Part 4: 운영 보안

    • WAS 서버 설정 - Flask/Gunicorn 보안
    • 로깅 - 침입 탐지
    • 백업 - 최후의 보루

    전체 구조

    보안 위협
├── 외부 공격
│   ├── SQL 인젝션 → DB 탈취/파괴
│   ├── XSS → 사용자 정보 탈취
│   └── CSRF → 권한 도용
│
├── 내부 위협
│   ├── 과도한 DB 권한
│   ├── 평문 비밀번호 저장
│   └── 민감 데이터 미암호화
│
└── 운영 실수
    ├── 디버그 모드 배포
    ├── 기본 비밀번호 사용
    └── 백업 미실시

    목차

    장 제목 핵심 내용
    1 보안이 왜 중요한가 실제 사례, 법적 책임
    2 SQL 인젝션 공격 공격 원리 이해
    3 SQL 인젝션 방어 Prepared Statement
    4 XSS 공격과 방어 이스케이프, CSP
    5 CSRF 공격과 방어 CSRF 토큰
    6 인증과 세션 관리 안전한 로그인
    7 비밀번호 안전하게 저장 bcrypt, 솔팅
    8 DB 사용자 권한 관리 GRANT, REVOKE
    9 DB 접근 제어 IP 제한, 방화벽
    10 민감 데이터 암호화 AES, 칼럼 암호화
    11 WAS 서버 보안 설정 Gunicorn, Nginx
    12 HTTPS 적용하기 Let's Encrypt
    13 로깅과 모니터링 침입 탐지
    14 백업과 복구 mysqldump, 자동화
    15 보안 체크리스트 최종 점검
