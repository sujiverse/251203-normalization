📙 2권: SQL 기초 편

데이터베이스 활용

    [10장] DBeaver 200% 활용

    DBeaver는 무료 데이터베이스 관리 도구예요. MySQL, PostgreSQL, SQLite 등 거의 모든 DB를 지원합니다. 이번 장에서는 업무 효율을 높이는 DBeaver 활용법을 배웁니다!

    💡 DBeaver vs MySQL Workbench

    • MySQL Workbench: MySQL 전용, 기능 풍부
    • DBeaver: 여러 DB 지원, 직관적 UI

    둘 다 좋은 도구예요. 여러분 상황에 맞는 걸 선택하세요!

    1. 자주 쓰는 쿼리 저장하기

    🎯 왜 쿼리를 저장할까요?

    매일 같은 리포트를 뽑는데 매번 쿼리를 새로 작성하시나요? 한 번 만든 쿼리는 저장해두고 재사용하면 시간이 절약됩니다!

    📖 쿼리 저장 방법

    1단계 쿼리 작성 후 Ctrl + S (저장)
    2단계 파일명 입력 (예: 월간매출리포트.sql)
    3단계 저장 위치 선택 (예: 내문서/SQL쿼리)

    🎭 비유로 이해하기

    쿼리 저장은 레시피를 적어두는 것과 같아요!

    • 매번 요리 순서를 외울 필요 없이
    • 레시피 노트를 펼치면 바로 요리 시작
    • SQL도 마찬가지! 저장해두면 바로 실행 가능
    💡 파일명 짓는 팁

    • 01_일일매출.sql - 번호로 순서 정리
    • 월간_치료사별실적.sql - 용도 명확히
    • 2024_연간보고서.sql - 연도 포함

    나중에 찾기 쉽게 명확한 이름을 붙이세요!

    2. 결과를 엑셀로 내보내기

    🎯 왜 엑셀로 내보낼까요?

    SQL 결과를 보고서로 만들거나, 차트를 그리거나, 추가 계산을 할 때 엑셀이 편리해요. DBeaver에서 클릭 몇 번이면 됩니다!

    📖 엑셀 내보내기 방법

    1단계 쿼리 실행 후 결과 확인
    2단계 결과창에서 우클릭
    3단계 Export Data... 선택
    4단계 Format에서 XLSX 선택
    5단계 저장 위치 지정 후 Start
    CSV 형식
    • 어디서든 열림
    • 파일 크기 작음
    • 서식 없음
    • 한글 깨짐 가능
    XLSX 형식
    • 엑셀 전용
    • 서식 유지 가능
    • 차트 추가 가능
    • 한글 문제없음

    ❓ CSV가 깨질 때는?

    CSV 파일을 엑셀에서 열었는데 한글이 깨진다면:

    1. 엑셀에서 데이터 → 텍스트/CSV에서 선택
    2. 인코딩을 UTF-8로 변경
    3. 로드

    3. 업무 속도를 높이는 단축키

    🎯 단축키를 왜 배울까요?

    마우스 클릭보다 키보드가 빠릅니다! 자주 쓰는 단축키 5개만 외워도 작업 속도가 확 올라가요.

    📖 DBeaver 필수 단축키

    단축키 기능 사용 상황
    Ctrl + Enter 쿼리 실행 쿼리 작성 후 바로 실행!
    Ctrl + Space 자동완성 테이블명, 컬럼명 자동완성
    Ctrl + S 저장 쿼리 파일 저장
    Ctrl + / 주석 처리 선택한 줄을 주석으로
    Ctrl + Shift + F SQL 정리 쿼리 자동 들여쓰기

    🎭 가장 많이 쓰는 조합

    쿼리 작성의 90%는 이 두 단축키로 해결됩니다:

    1. Ctrl + Space: 테이블, 컬럼명 자동완성
    2. Ctrl + Enter: 실행!

    이 두 개만 외워도 충분해요!

    💡 부분 실행 팁

    여러 쿼리가 한 파일에 있을 때:
    • 실행할 쿼리만 드래그해서 선택
    • Ctrl + Enter
    → 선택한 부분만 실행됩니다!

    4. 자동완성으로 오타 방지

    🎯 자동완성의 장점

    테이블명이나 컬럼명을 직접 타이핑하면 오타가 나기 쉬워요. 자동완성을 쓰면 오타도 방지하고 타이핑 시간도 줄어듭니다!

    📖 자동완성 사용법

    -- 테이블명 자동완성
SELECT * FROM pa        -- 여기서 Ctrl+Space 누르면
                        -- patients, patient_diagnoses 등 목록 표시!

-- 컬럼명 자동완성
SELECT p.               -- 점(.) 찍으면 자동으로 컬럼 목록 표시!
    patient_id,
    name,
    phone
FROM patients p;
    pa + Ctrl+Space
    "pa"로 시작하는 테이블 목록이 나타나요!

    • patients
    • patient_diagnoses
    • patient_insurance

    화살표로 선택 후 Enter!
    p. (별칭 + 점)
    테이블에 별칭(alias)을 붙이고 점을 찍으면 해당 테이블의 모든 컬럼 목록이 자동으로 나타나요!

    일일이 컬럼명을 외울 필요가 없어요!

    ❓ 자동완성이 안 될 때는?

    자동완성이 작동하지 않는다면:

    • 데이터베이스에 연결이 되어 있는지 확인
    • 좌측 패널에서 테이블 목록이 새로고침 됐는지 확인
    • DBeaver가 메타데이터를 읽는 중일 수 있음 (잠시 대기)

    5. 테이블 관계도 (ER 다이어그램)

    🎯 ER 다이어그램이 뭔가요?

    테이블들이 어떻게 연결되어 있는지 그림으로 보여주는 거예요. JOIN을 작성할 때 어떤 컬럼으로 연결해야 하는지 한눈에 알 수 있습니다!

    📖 ER 다이어그램 보는 방법

    1단계 좌측 Database Navigator에서 데이터베이스 선택
    2단계 데이터베이스 우클릭 → View Diagram
    3단계 테이블 간 선(관계) 확인

    🎭 ER 다이어그램 읽는 법

    기호의미예시
    ─────1:1 관계환자 - 환자상세
    ───<1:N 관계환자 - 치료기록
    >───<N:M 관계환자 - 진단 (중간테이블 필요)

    선을 따라가면 어떤 컬럼으로 JOIN해야 하는지 알 수 있어요!

    💡 JOIN 작성 시 활용법

    "환자와 치료 기록을 연결하려면?"
    1. ER 다이어그램에서 두 테이블 찾기
    2. 연결하는 선 확인 (보통 화살표가 가리키는 쪽이 외래키)
    3. ON patients.patient_id = treatment_sessions.patient_id

    📝 이번 장 핵심 정리

    기능 방법 효과
    쿼리 저장 Ctrl + S 재사용으로 시간 절약
    엑셀 내보내기 우클릭 → Export 보고서 작성 용이
    쿼리 실행 Ctrl + Enter 빠른 실행
    자동완성 Ctrl + Space 오타 방지, 속도 향상
    ER 다이어그램 View Diagram 테이블 관계 파악
