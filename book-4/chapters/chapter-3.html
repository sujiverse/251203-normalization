    1. Python 설치

    Windows

    1. python.org 접속
    2. Downloads → Windows 클릭
    3. 최신 버전 다운로드 (3.11 이상 권장)
    4. 중요! 설치 시 "Add Python to PATH" 체크
    5. Install Now 클릭

    Mac

    1. python.org 접속
    2. Downloads → macOS 클릭
    3. 최신 버전 다운로드
    4. 설치 진행

    설치 확인

    # 터미널/명령 프롬프트에서
python --version
# 또는
python3 --version

# 결과 예시
Python 3.11.5
    ⚠️ python이 안 되면 python3로!
    Mac에서는 python 대신 python3를 사용해야 할 수 있습니다.

    2. VS Code 설치

    1. code.visualstudio.com 접속
    2. Download for Windows/Mac 클릭
    3. 설치 파일 실행
    4. 기본 옵션으로 설치

    추천 확장 프로그램

    VS Code 실행 후 왼쪽 Extensions 아이콘 클릭, 검색해서 설치:

    • Python - Python 지원
    • Pylance - Python 자동완성
    • Live Server - HTML 미리보기

    3. 프로젝트 폴더 만들기

    작업할 폴더를 만듭니다.

    # 원하는 위치에 폴더 생성
mkdir healing-hands-app
cd healing-hands-app

    폴더 구조 (최종 모습)

    healing-hands-app/
├── app.py              ← Flask 서버 (백엔드)
├── templates/          ← HTML 파일들
│   ├── index.html
│   └── reservations.html
├── static/             ← CSS, JS, 이미지
│   ├── css/
│   ├── js/
│   └── lib/            ← Luckysheet 등 라이브러리
└── venv/               ← Python 가상환경

    4. 가상환경 만들기

    프로젝트별로 독립된 Python 환경을 만듭니다.

    # 가상환경 생성
python -m venv venv

# 가상환경 활성화 (Windows)
venv\Scripts\activate

# 가상환경 활성화 (Mac/Linux)
source venv/bin/activate

# 활성화되면 프롬프트에 (venv) 표시됨
(venv) C:\healing-hands-app>
    💡 왜 가상환경을 쓰나요?
    프로젝트마다 필요한 라이브러리 버전이 다를 수 있습니다. 가상환경을 쓰면 프로젝트 간 충돌을 방지합니다.

    5. Flask 설치

    가상환경이 활성화된 상태에서:

    # Flask 설치
pip install flask

# MySQL 연결 라이브러리도 설치
pip install flask-mysqldb
# 또는
pip install pymysql

    설치 확인

    pip list
# Flask, pymysql 등이 보이면 성공!

    6. 테스트: Hello World

    모든 게 잘 설치됐는지 확인해봅시다.

    app.py 파일 생성

    # app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return '안녕하세요! Flask가 작동합니다!'

if __name__ == '__main__':
    app.run(debug=True)

    실행

    python app.py

    확인

    웹 브라우저에서 http://127.0.0.1:5000 접속

    "안녕하세요! Flask가 작동합니다!" 가 보이면 성공!

    💡 서버 종료 방법
    터미널에서 Ctrl + C를 누르면 Flask 서버가 종료됩니다.
