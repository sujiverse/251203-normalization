웹 앱 만들기 (4권)

    우리가 만들 것 미리보기

    완성된 모습부터 봅시다

    지금부터 뭘 만들지 모르고 따라가면 지루하고 힘듭니다. 먼저 완성된 결과물을 보면서 "아, 이걸 만드는구나!" 하고 목표를 명확히 합시다.

    화면 1: 예약 현황 시트

    엑셀처럼 생긴 예약 관리 화면입니다.

    환자명 12/1(월) 12/2(화) 12/3(수) 12/4(목) 12/5(금)
    김철수 10:00 박치료 14:00 김치료 11:00 박치료
    이영희 11:00 이치료 10:00 이치료
    박민수 15:00 김치료 15:00 박치료 16:00 김치료

    특징

    • 첫 열 고정 - 환자명은 항상 보임
    • 가로 스크롤 - 날짜가 많아도 OK
    • 색상 구분 - 치료사별로 다른 색
    • 셀 클릭 - 예약 등록/수정 가능
    • 실시간 저장 - 편집하면 DB에 바로 반영

    화면 2: 환자 관리

    환자 목록을 보고, 추가/수정할 수 있는 화면입니다.

    이름 연락처 생년월일 등록일 관리
    김철수 010-1234-5678 1985-03-15 2024-01-10 [수정] [삭제]
    이영희 010-2345-6789 1990-07-22 2024-02-05 [수정] [삭제]

    화면 3: 매출 현황

    일별/월별 매출을 한눈에 볼 수 있습니다.

    날짜 치료 건수 매출 수납완료 미수금
    2024-12-01 15건 450,000원 400,000원 50,000원
    2024-12-02 12건 360,000원 360,000원 0원

    시스템 구조

    사용자 (웹 브라우저)
    │
    │  ① 셀 편집
    ▼
프론트엔드 (Luckysheet)
    │
    │  ② API 호출 (fetch)
    ▼
백엔드 (Flask)
    │
    │  ③ SQL 실행
    ▼
데이터베이스 (MySQL)
    │
    │  ④ 결과 반환
    ▼
    ... (역순으로 화면에 표시)

    이 흐름을 이해하는 게 4권의 핵심입니다. 각 단계를 하나씩 만들어가면서 배웁니다.

    Metabase와 뭐가 다른가요?

    항목 Metabase (3권) 웹 앱 (4권)
    데이터 조회 O O
    데이터 수정 X (조회만) O
    커스텀 UI 제한적 자유롭게
    엑셀 스타일 X O
    개발 난이도 쉬움 중간
    💡 Metabase도 계속 사용하세요!
    분석과 리포팅은 Metabase가 더 편합니다. 웹 앱은 "입력과 편집"에 특화된 도구로 함께 사용하면 좋습니다.

    이번 장 정리

    • 예약 현황을 엑셀처럼 보고 편집하는 웹 앱을 만듭니다
    • 프론트엔드 → 백엔드 → DB의 흐름을 이해합니다
    • Metabase와 다르게 "데이터 수정"이 가능합니다
    다음 장에서는 이 시스템의 구조를 더 자세히 살펴봅니다.
