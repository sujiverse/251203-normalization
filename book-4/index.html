웹 앱 만들기 (4권)

    이 책의 사용법

    드디어 웹 앱을 만듭니다

    1권에서는 데이터베이스 개념을, 2권에서는 SQL 문법을, 3권에서는 실제 시스템 구축과 Metabase 대시보드를 배웠습니다.

    그런데 Metabase만으로는 부족하지 않으셨나요?

    예약을 엑셀처럼 한눈에 보고 싶고, 셀을 클릭해서 바로 수정하고 싶고, 내 입맛대로 화면을 꾸미고 싶지 않으셨나요?

    4권에서는 DB와 연동되는 진짜 웹 앱을 만듭니다. 프론트엔드, 백엔드, 데이터베이스 - 풀스택 개발의 기초를 배우고, 엑셀처럼 사용할 수 있는 예약 관리 시스템을 완성합니다.

    이 책에서 만들 것

    엑셀 스타일 예약 관리 시스템

    12/1(월) 12/2(화) 12/3(수) 12/4(목)
    김철수 10:00 14:00
    이영희 11:00 10:00
    박민수 15:00 15:00 16:00
    • 첫 번째 열(환자명)은 고정
    • 날짜는 가로로 스크롤
    • 셀을 클릭해서 예약 등록/수정
    • 변경하면 DB에 자동 저장

    기술 스택

    영역 기술 역할
    프론트엔드 HTML, CSS, JavaScript 화면 구성
    스프레드시트 Luckysheet 엑셀 UI
    백엔드 Python, Flask API 서버
    데이터베이스 MySQL 데이터 저장
    💡 3권의 DB를 그대로 사용합니다
    3권에서 만든 healing_hands 데이터베이스를 그대로 활용합니다. 3권을 완료하지 않았다면, 부록의 SQL 스크립트로 빠르게 세팅할 수 있습니다.

    전체 구조

    ├── 프론트엔드 (브라우저)
│   ├── HTML - 화면 구조
│   ├── CSS - 스타일
│   ├── JavaScript - 동작
│   └── Luckysheet - 엑셀 UI
│
├── 백엔드 (Flask 서버)
│   ├── API 엔드포인트
│   └── DB 연결
│
└── 데이터베이스 (MySQL)
    └── healing_hands DB

    사용자가 화면에서 셀을 편집하면 → JavaScript가 API를 호출하고 → Flask가 요청을 받아서 → MySQL에 저장합니다.
