    데이터베이스 & SQL 용어

    이 책에서 사용된 용어들을 알파벳/가나다순으로 정리했습니다. 모르는 용어가 나오면 여기서 찾아보세요.

    A ~ Z

    용어설명
    AUTO_INCREMENT 자동 증가. 새 행이 추가될 때마다 자동으로 1씩 증가하는 숫자를 생성합니다. 주로 기본키에 사용됩니다.
    session_id INT AUTO_INCREMENT PRIMARY KEY
    CHAR 고정 길이 문자열 타입. CHAR(1)은 정확히 1글자, CHAR(10)은 정확히 10글자를 저장합니다. 항상 고정된 길이가 필요할 때 사용합니다.
    gender CHAR(1) -- 'M' 또는 'F'
    CHECK 값의 유효성을 검사하는 제약조건. 조건에 맞지 않는 값은 입력이 거부됩니다.
    CHECK (gender IN ('M', 'F'))
    COALESCE NULL이 아닌 첫 번째 값을 반환하는 함수. NULL 대신 기본값을 표시할 때 유용합니다.
    COALESCE(memo, '없음') -- memo가 NULL이면 '없음' 반환
    COMMIT 트랜잭션에서 변경 사항을 확정(저장)하는 명령. COMMIT 후에는 변경을 취소할 수 없습니다.
    COUNT 행의 개수를 세는 집계 함수.
    SELECT COUNT(*) FROM patients -- 전체 환자 수
    CREATE 데이터베이스, 테이블, 인덱스 등을 생성하는 명령.
    CREATE TABLE patients (...)
    CRUD Create(생성), Read(조회), Update(수정), Delete(삭제)의 약자. 데이터베이스의 기본 4가지 작업을 의미합니다.
    CURDATE() 현재 날짜를 반환하는 MySQL 함수.
    WHERE session_date = CURDATE() -- 오늘 날짜
    DATE 날짜를 저장하는 데이터 타입. 'YYYY-MM-DD' 형식입니다.
    birth_date DATE -- 예: '1985-03-15'
    DATETIME 날짜와 시간을 함께 저장하는 타입. 'YYYY-MM-DD HH:MM:SS' 형식입니다.
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    DELETE 데이터를 삭제하는 명령. WHERE 없이 사용하면 전체 데이터가 삭제되니 주의!
    DELETE FROM patients WHERE patient_id = 'P001'
    DESC / DESCRIBE 테이블 구조를 확인하는 명령.
    DESCRIBE patients -- 컬럼 목록과 타입 표시
    DISTINCT 중복을 제거하고 고유한 값만 반환합니다.
    SELECT DISTINCT therapist_id FROM treatment_sessions
    Docker 컨테이너 기반 가상화 플랫폼. 이 책에서는 Metabase를 실행하는 데 사용합니다.
    DROP 데이터베이스, 테이블 등을 완전히 삭제하는 명령. 복구 불가능하니 신중하게!
    DROP TABLE patients -- 테이블 완전 삭제
    ER 다이어그램 Entity-Relationship Diagram. 테이블 간의 관계를 시각적으로 표현한 그림입니다.
    EXPLAIN 쿼리 실행 계획을 보여주는 명령. 쿼리 최적화에 사용합니다.
    EXPLAIN SELECT * FROM patients WHERE name = '강민준'
    FK (Foreign Key) 외래키. 다른 테이블의 기본키를 참조하여 테이블 간 관계를 만듭니다.
    FOREIGN KEY (patient_id) REFERENCES patients(patient_id)
    GROUP BY 데이터를 그룹으로 묶어서 집계할 때 사용합니다.
    SELECT therapist_id, COUNT(*) FROM treatment_sessions GROUP BY therapist_id
    HAVING GROUP BY 결과에 조건을 거는 절. WHERE는 그룹화 전, HAVING은 그룹화 후 필터링입니다.
    GROUP BY patient_id HAVING COUNT(*) > 5
    INDEX 검색 속도를 높이기 위한 데이터 구조. 책의 색인과 같은 역할입니다.
    CREATE INDEX idx_name ON patients(name)
    INSERT 새 데이터를 추가하는 명령.
    INSERT INTO patients (patient_id, name) VALUES ('P001', '강민준')
    INT 정수를 저장하는 데이터 타입. 약 -21억 ~ +21억 범위입니다.
    JOIN 두 개 이상의 테이블을 연결하여 조회하는 방법.
    SELECT * FROM patients p JOIN treatment_sessions ts ON p.patient_id = ts.patient_id
    LAST_INSERT_ID() 가장 최근에 AUTO_INCREMENT로 생성된 값을 반환합니다.
    LEFT JOIN 왼쪽 테이블의 모든 행을 포함하고, 오른쪽 테이블에서 일치하는 행을 연결합니다. 일치하지 않으면 NULL이 됩니다.
    LIKE 패턴 매칭 검색. %는 아무 문자열, _는 한 글자를 의미합니다.
    WHERE name LIKE '김%' -- '김'으로 시작하는 이름
    LIMIT 결과 행 수를 제한합니다.
    SELECT * FROM patients LIMIT 10 -- 최대 10개만
    Metabase 오픈소스 BI(Business Intelligence) 도구. SQL 없이도 데이터를 시각화하고 대시보드를 만들 수 있습니다.
    MySQL 가장 널리 사용되는 오픈소스 관계형 데이터베이스 시스템입니다.
    NOT NULL NULL 값을 허용하지 않는 제약조건. 반드시 값이 있어야 합니다.
    name VARCHAR(50) NOT NULL
    NULL 값이 없음을 나타내는 특수 값. 0이나 빈 문자열과는 다릅니다.
    ORDER BY 결과를 정렬합니다. ASC는 오름차순, DESC는 내림차순입니다.
    ORDER BY session_date DESC -- 최신순
    PK (Primary Key) 기본키. 테이블에서 각 행을 고유하게 식별하는 컬럼입니다. 중복과 NULL이 불가능합니다.
    Query 쿼리. 데이터베이스에 보내는 요청/명령입니다. SQL 문장 자체를 의미하기도 합니다.
    ROLLBACK 트랜잭션에서 변경 사항을 취소하고 이전 상태로 되돌리는 명령입니다.
    Schema 스키마. 데이터베이스의 구조(테이블, 컬럼, 관계 등)를 정의한 것입니다.
    SELECT 데이터를 조회하는 명령. SQL에서 가장 많이 사용됩니다.
    SELECT * FROM patients WHERE is_active = 'Y'
    SQL Structured Query Language. 데이터베이스를 다루는 표준 언어입니다.
    SUM 합계를 계산하는 집계 함수.
    SELECT SUM(total_amount) FROM billings
    TEXT 긴 문자열을 저장하는 데이터 타입. 메모, 설명 등에 사용합니다.
    TIME 시간을 저장하는 데이터 타입. 'HH:MM:SS' 형식입니다.
    start_time TIME -- 예: '10:30:00'
    Transaction 트랜잭션. 여러 작업을 하나의 단위로 묶어서 처리하는 것. 모두 성공하거나 모두 실패합니다.
    UNIQUE 중복 값을 허용하지 않는 제약조건.
    UNIQUE (name, birth_date, phone)
    UPDATE 기존 데이터를 수정하는 명령.
    UPDATE patients SET phone = '010-1234-5678' WHERE patient_id = 'P001'
    VARCHAR 가변 길이 문자열 타입. VARCHAR(50)은 최대 50글자까지 저장 가능합니다.
    name VARCHAR(50) NOT NULL
    WHERE 조건을 지정하는 절. 조건에 맞는 행만 선택됩니다.
    WHERE session_date = '2024-03-25'

    ㄱ ~ ㅎ

    용어설명
    기본키 Primary Key(PK). 테이블에서 각 행을 고유하게 식별하는 컬럼. 중복 불가, NULL 불가.
    노쇼 (No-Show) 예약했지만 연락 없이 나타나지 않는 것. 클리닉 운영에서 큰 손실입니다.
    다대다 관계 N:M 관계. 양쪽 테이블 모두 여러 개의 연결이 가능한 관계. 중간 테이블(연결 테이블)이 필요합니다. 예: 환자-진단
    대시보드 여러 차트와 지표를 한 화면에 모아놓은 것. 현황을 한눈에 파악할 수 있습니다.
    데이터 무결성 데이터의 정확성과 일관성을 유지하는 것. 제약조건으로 보장합니다.
    데이터베이스 체계적으로 구조화된 데이터의 집합. 여러 테이블로 구성됩니다.
    레코드 (Record) 테이블의 한 행(Row). 하나의 데이터 단위를 의미합니다. 예: 환자 한 명의 정보
    미수금 아직 받지 못한 돈. 치료는 완료했지만 수납이 안 된 상태입니다.
    백업 데이터를 복사해서 안전한 곳에 보관하는 것. 문제 발생 시 복원에 사용합니다.
    복원 백업 파일을 사용해서 데이터를 원래 상태로 되돌리는 것.
    비정규화 정규화의 반대. 성능 향상을 위해 의도적으로 중복을 허용하는 것.
    수가 의료 서비스의 가격. 치료 항목별로 정해진 금액입니다.
    스키마 데이터베이스의 구조를 정의한 것. 테이블, 컬럼, 관계 등의 설계도입니다.
    시각화 데이터를 차트, 그래프 등으로 시각적으로 표현하는 것.
    실손보험 실제 발생한 의료비를 보상해주는 민간 보험. 도수치료 비용 청구에 사용됩니다.
    외래키 Foreign Key(FK). 다른 테이블의 기본키를 참조하는 컬럼. 테이블 간 관계를 만듭니다.
    이탈 환자가 더 이상 방문하지 않는 것. 30일 이상 미방문 시 이탈 위험으로 분류합니다.
    인덱스 검색 속도를 높이기 위한 자료구조. 자주 검색하는 컬럼에 생성합니다.
    인센티브 성과에 따른 추가 보상. 이 책에서는 치료사 실적에 따른 보너스를 계산합니다.
    일대다 관계 1:N 관계. 한쪽이 여러 개와 연결되는 관계. 예: 환자 1명 - 치료 여러 건
    일대일 관계 1:1 관계. 양쪽이 하나씩만 연결되는 관계. 예: 치료 세션 1건 - SOAP 노트 1개
    정규화 데이터 중복을 제거하고 무결성을 높이기 위해 테이블을 분리하는 과정입니다.
    제약조건 데이터의 무결성을 보장하기 위한 규칙. PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, NOT NULL 등이 있습니다.
    조인 두 개 이상의 테이블을 연결해서 조회하는 것. INNER JOIN, LEFT JOIN, RIGHT JOIN 등이 있습니다.
    집계 함수 여러 행의 값을 계산하는 함수. COUNT, SUM, AVG, MAX, MIN 등이 있습니다.
    질문 (Question) Metabase 용어. 하나의 데이터 조회 또는 분석 단위입니다.
    청구 치료 비용을 환자나 보험사에 요청하는 것.
    초진 환자의 첫 방문 또는 첫 진료. 신규 환자 상담을 의미합니다.
    컬럼 (Column) 테이블의 세로 열. 데이터의 속성을 나타냅니다. 예: name, phone, birth_date
    컬렉션 (Collection) Metabase 용어. 질문과 대시보드를 정리하는 폴더입니다.
    쿼리 데이터베이스에 보내는 명령. SQL 문장을 의미합니다.
    테이블 데이터를 행과 열로 구성한 표. 데이터베이스의 기본 저장 단위입니다.
    트랜잭션 여러 작업을 하나의 단위로 묶은 것. 모두 성공(COMMIT)하거나 모두 취소(ROLLBACK)됩니다.
    필드 (Field) 컬럼(Column)과 같은 의미로 사용됩니다.
    해피콜 일정 기간 방문하지 않은 환자에게 안부를 묻고 재방문을 유도하는 전화입니다.
    행 (Row) 테이블의 가로 줄. 하나의 데이터 레코드를 의미합니다.

    물리치료 용어

    용어설명
    ROM Range of Motion. 관절가동범위. 관절이 움직일 수 있는 범위를 측정합니다.
    SLR Straight Leg Raise. 하지직거상검사. 디스크 탈출 여부를 확인하는 검사입니다.
    SOAP Subjective(주관적), Objective(객관적), Assessment(평가), Plan(계획). 의료 기록 형식입니다.
    VDT 증후군 Visual Display Terminal 증후군. 컴퓨터 작업으로 인한 목, 어깨, 눈의 통증입니다.
    도수치료 치료사의 손으로 직접 시행하는 치료. 관절가동술, 근막이완 등이 포함됩니다.
    슈로스 Schroth. 측만증 교정을 위한 특수 운동치료법입니다.
    오십견 유착성 관절낭염. 어깨가 굳어서 움직이기 어려운 상태입니다.
    운동치료 재활 운동을 통해 치료하는 방법. 근력 강화, 유연성 향상 등이 목적입니다.
    측만증 척추가 옆으로 휜 상태. Cobb angle로 각도를 측정합니다.
    체외충격파 ESWT. 충격파를 이용한 치료. 건염, 석회화 등에 사용됩니다.
    회전근개 어깨를 감싸고 있는 4개의 근육과 힘줄. 손상되면 어깨 통증이 발생합니다.

    자주 사용하는 MySQL 함수

    함수설명예시
    CURDATE()오늘 날짜'2024-03-25'
    CURTIME()현재 시간'14:30:00'
    NOW()현재 날짜와 시간'2024-03-25 14:30:00'
    DATE_FORMAT(date, format)날짜 형식 변환DATE_FORMAT(date, '%Y-%m') → '2024-03'
    DATEDIFF(d1, d2)날짜 차이 (일)DATEDIFF('2024-03-25', '2024-03-01') → 24
    DATE_SUB(date, INTERVAL n unit)날짜 빼기DATE_SUB(CURDATE(), INTERVAL 7 DAY)
    TIMESTAMPDIFF(unit, d1, d2)시간 차이TIMESTAMPDIFF(YEAR, birth_date, CURDATE())
    CONCAT(s1, s2, ...)문자열 연결CONCAT(name, ' (', phone, ')')
    COALESCE(v1, v2, ...)첫 번째 NULL 아닌 값COALESCE(memo, '없음')
    IFNULL(v1, v2)NULL이면 대체값IFNULL(memo, '없음')
    ROUND(n, d)반올림ROUND(123.456, 1) → 123.5
    FORMAT(n, d)천 단위 구분FORMAT(1234567, 0) → '1,234,567'
    GREATEST(v1, v2, ...)가장 큰 값GREATEST(0, amount - 100) → 음수면 0
