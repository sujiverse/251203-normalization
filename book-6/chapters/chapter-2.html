    1부: 연구와 데이터의 기초

    [2장] 연구 설계의 기본 구조

    이 장에서는 연구 질문에서 데이터 구조까지 어떻게 연결되는지를 배웁니다. 좋은 연구 설계가 좋은 데이터 구조의 출발점입니다.

    1. 연구 질문에서 시작하기

    모든 연구는 질문에서 시작합니다. 연구 질문(Research Question)은 연구를 통해 답하고자 하는 구체적인 물음입니다.

    좋은 연구 질문의 조건 (FINER)

    • Feasible (실현 가능): 주어진 자원으로 수행 가능
    • Interesting (흥미로움): 연구자와 학계의 관심사
    • Novel (새로움): 기존 연구와 차별점
    • Ethical (윤리적): 윤리적으로 수행 가능
    • Relevant (관련성): 학문적/실용적 가치

    연구 질문의 유형

    유형질문 형태예시필요한 변수
    기술적 "~은 어떠한가?" 한국 성인의 평균 수면 시간은? 수면 시간 (1개)
    관계적 "~과 ~의 관계는?" 운동량과 우울감의 관계는? 운동량, 우울감 (2개)
    비교적 "~과 ~은 다른가?" 남녀의 수면 시간 차이는? 성별, 수면 시간 (2개)
    인과적 "~이 ~에 영향을?" 수면 교육이 수면의 질을 높이나? 처치, 결과, 통제변수 (3개+)

    연구 질문 → 변수 도출

    연구 질문을 분석하면 필요한 변수가 드러납니다:

    질문: "물리치료가 만성 요통 환자의 통증과 기능에 미치는 효과는?"

    • 독립변수: 물리치료 여부 (실험군/대조군)
    • 종속변수: 통증 점수, 기능 점수
    • 대상: 만성 요통 환자
    • 암묵적 통제변수: 나이, 성별, 통증 기간 등

    2. 모집단과 표본

    연구는 특정 집단에 대한 결론을 도출합니다. 그 집단이 모집단입니다.

    모집단 (Population)

    연구 결과를 적용하고자 하는 전체 집단입니다.

    • 목표 모집단(Target Population): 결론을 일반화하고 싶은 집단
      예: 한국의 모든 만성 요통 환자
    • 접근 가능 모집단(Accessible Population): 실제 접근 가능한 집단
      예: 서울 소재 3개 병원의 만성 요통 환자

    표본 (Sample)

    모집단에서 실제로 연구에 참여하는 일부입니다.

    • 표본은 모집단을 대표해야 합니다
    • 표본 크기는 통계적 검정력에 영향을 미칩니다
    • 편향된 표본은 일반화를 제한합니다

    표본 추출 방법

    방법설명장단점
    단순 무작위 추출 모집단에서 무작위로 선택 대표성 높음, 실행 어려움
    층화 추출 하위 집단별로 무작위 추출 하위 집단 대표성 확보
    군집 추출 집단 단위로 추출 후 전수 조사 효율적, 대표성 낮을 수 있음
    편의 추출 접근 가능한 대상 선택 쉬움, 편향 위험 높음

    DB 설계: 표본 추출 정보 저장

    CREATE TABLE sampling_info (
    id INTEGER PRIMARY KEY,
    method TEXT,  -- 'random', 'stratified', 'convenience' 등
    target_population TEXT,
    accessible_population TEXT,
    sampling_date DATE,
    total_invited INTEGER,
    total_enrolled INTEGER
);

CREATE TABLE participants (
    id INTEGER PRIMARY KEY,
    sampling_info_id INTEGER REFERENCES sampling_info(id),
    stratum TEXT,  -- 층화 추출 시 하위 집단
    enrolled_date DATE
);

    3. 가설 설정

    가설(Hypothesis)은 연구 질문에 대한 잠정적 답변입니다. 통계적 검정의 대상이 됩니다.

    귀무가설 (H₀, Null Hypothesis)

    차이나 효과가 없다는 가설입니다.

    • "두 집단의 평균에 차이가 없다"
    • "두 변수 간 상관이 없다"
    • "처치의 효과가 없다"

    통계 검정은 귀무가설을 기각하거나 기각하지 못함으로 결론짓습니다. 귀무가설을 "증명"하지는 않습니다.

    대립가설 (H₁, Alternative Hypothesis)

    차이나 효과가 있다는 가설입니다. 연구자가 검증하고자 하는 가설입니다.

    비방향성 가설 (양측 검정):

    • "두 집단의 평균이 다르다" (크거나 작거나)
    • H₁: μ₁ ≠ μ₂

    방향성 가설 (단측 검정):

    • "실험군의 평균이 대조군보다 높다"
    • H₁: μ₁ > μ₂

    가설과 오류

    H₀가 참H₀가 거짓
    H₀ 기각 1종 오류 (α)
    없는 효과를 있다고 함    		 올바른 결정
    (검정력 = 1-β)
    H₀ 기각 못함 올바른 결정 2종 오류 (β)
    있는 효과를 못 찾음

    유의수준(α): 1종 오류를 허용하는 확률. 보통 0.05 사용

    검정력(Power): 실제 효과가 있을 때 이를 탐지할 확률. 보통 0.80 이상 권장

    4. 연구 설계 → DB 구조 매핑

    연구의 각 요소는 데이터베이스 구조로 변환됩니다.

    연구 요소와 DB 요소 대응

    연구 요소DB 요소예시
    참가자/피험자 participants 테이블의 행 id, age, gender
    변수 테이블의 열(column) pain_score, function_score
    측정값 셀(cell)의 값 75, 82
    측정 시점 waves/timepoints 테이블 pre, post, follow_up
    실험 조건 conditions 테이블 treatment, control
    척도/검사 instruments 테이블 VAS, ODI, SF-36

    기본 연구 DB 스키마

    -- 연구 정보
CREATE TABLE studies (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    start_date DATE,
    end_date DATE,
    design_type TEXT  -- 'cross-sectional', 'longitudinal', 'experimental'
);

-- 참가자 (시간 불변 정보)
CREATE TABLE participants (
    id INTEGER PRIMARY KEY,
    study_id INTEGER REFERENCES studies(id),
    enrollment_date DATE,
    birth_year INTEGER,
    gender TEXT CHECK(gender IN ('M', 'F', 'Other'))
);

-- 측정 도구
CREATE TABLE instruments (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    abbreviation TEXT,
    min_score REAL,
    max_score REAL,
    higher_is_better BOOLEAN
);

-- 측정값
CREATE TABLE measurements (
    id INTEGER PRIMARY KEY,
    participant_id INTEGER REFERENCES participants(id),
    instrument_id INTEGER REFERENCES instruments(id),
    time_point TEXT,
    measure_date DATE,
    score REAL
);

    설계 원칙

    • 참가자 정보와 측정값 분리: 시간에 따라 변하지 않는 정보(성별, 출생연도)와 반복 측정값을 다른 테이블에 저장
    • 측정 도구 메타데이터 분리: 검사 도구의 정보(점수 범위, 해석 방향)를 별도 테이블에 저장
    • Long 형식 유지: 각 측정은 별도의 행으로 저장

    5. 좋은 연구 설계의 조건

    연구의 질은 타당도(Validity)와 신뢰도(Reliability)로 평가합니다.

    내적 타당도 (Internal Validity)

    독립변수가 종속변수에 실제로 영향을 미쳤는지에 대한 확신의 정도입니다.

    위협 요인:

    • 역사(History): 연구 기간 중 외부 사건의 영향
    • 성숙(Maturation): 시간 경과에 따른 자연적 변화
    • 검사 효과(Testing): 사전 검사가 사후 검사에 영향
    • 도구 변화(Instrumentation): 측정 도구의 변화
    • 통계적 회귀(Regression): 극단값이 평균으로 회귀
    • 선택 편향(Selection): 집단 간 사전 차이
    • 탈락(Attrition): 참가자 이탈로 인한 편향

    통제 방법:

    • 무선 할당(Random Assignment)
    • 통제 집단 사용
    • 맹검(Blinding)

    외적 타당도 (External Validity)

    연구 결과를 다른 상황, 다른 집단에 일반화할 수 있는 정도입니다.

    위협 요인:

    • 표본 특수성: 특정 집단에서만 적용 가능
    • 상황 특수성: 특정 환경에서만 적용 가능
    • 시간 특수성: 특정 시기에만 적용 가능
    • 실험실 인공성: 실제 상황과 다른 실험 조건

    향상 방법:

    • 대표성 있는 표본 추출
    • 다양한 환경에서 반복 연구
    • 현장 연구(Field Study)

    신뢰도 (Reliability)

    측정이 일관되게 반복되는 정도입니다.

    유형측정 방법기준
    검사-재검사 신뢰도 같은 검사를 시간 간격 두고 반복, 상관 계산 r ≥ 0.70
    내적 일관성 Cronbach's α 계산 α ≥ 0.70
    반분 신뢰도 검사를 반으로 나누어 상관 r ≥ 0.70
    평가자 간 신뢰도 여러 평가자의 일치도 (ICC, Kappa) ICC ≥ 0.75, κ ≥ 0.60

    타당도와 신뢰도의 관계

    신뢰도는 타당도의 필요조건입니다.

    • 신뢰도가 낮으면 타당도도 낮습니다 (측정이 일관되지 않으면 무엇을 측정하는지 알 수 없음)
    • 그러나 신뢰도가 높다고 타당도가 보장되지는 않습니다 (일관되게 엉뚱한 것을 측정할 수 있음)

    DB 설계: 신뢰도 정보 저장

    -- 측정 도구 신뢰도 정보
CREATE TABLE instrument_reliability (
    id INTEGER PRIMARY KEY,
    instrument_id INTEGER REFERENCES instruments(id),
    reliability_type TEXT,  -- 'test_retest', 'internal', 'inter_rater'
    coefficient REAL,
    sample_size INTEGER,
    reference TEXT
);

-- 평가자 간 신뢰도를 위한 다중 평가
CREATE TABLE ratings (
    id INTEGER PRIMARY KEY,
    participant_id INTEGER REFERENCES participants(id),
    instrument_id INTEGER REFERENCES instruments(id),
    rater_id INTEGER,
    score REAL
);
