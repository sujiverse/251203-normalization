    2부: 변수 유형과 저장 원칙

    [5장] 척도의 4가지 수준

    이 장에서는 측정 척도의 4가지 수준을 배웁니다. 척도 수준에 따라 가능한 분석 방법이 달라지므로, 이 분류를 정확히 이해하는 것이 중요합니다.

    1. 명목 척도 (Nominal Scale)

    명목 척도는 단순히 분류만 하는 척도입니다. 범주 간에 순서나 크기가 없습니다.

    특성

    • 분류 기능만 있음: 같다/다르다만 구분
    • 순서 없음: 범주 간 크고 작음이 없음
    • 숫자는 레이블일 뿐: 1=남, 2=여에서 2가 1보다 큰 것이 아님

    예시

    변수범주숫자 코딩
    성별남성, 여성1, 2 (또는 M, F)
    혈액형A, B, O, AB1, 2, 3, 4
    결혼 상태미혼, 기혼, 이혼1, 2, 3
    진단명요통, 경추통, 어깨통LBP, NP, SP

    가능한 연산과 분석

    • 연산: = (같다), ≠ (다르다)
    • 대표값: 최빈값(Mode)
    • 분석:
      • 빈도 분석 (Frequency)
      • 카이제곱 검정 (χ²)
      • 피셔 정확 검정

    DB 저장:

    -- 방법 1: 텍스트로 저장
gender TEXT CHECK(gender IN ('M', 'F'))

-- 방법 2: 정수 코드 + 별도 레이블 테이블
gender_code INTEGER,
FOREIGN KEY (gender_code) REFERENCES gender_labels(code)

-- 레이블 테이블
CREATE TABLE gender_labels (
    code INTEGER PRIMARY KEY,
    label TEXT
);
INSERT INTO gender_labels VALUES (1, '남성'), (2, '여성');

    2. 서열 척도 (Ordinal Scale)

    서열 척도는 범주 간에 순서가 있지만, 간격이 균등하지 않습니다.

    특성

    • 순서 있음: 크다/작다 비교 가능
    • 간격 불균등: 1등과 2등의 차이 ≠ 2등과 3등의 차이
    • 덧셈/뺄셈 의미 없음: 순위 간 거리를 알 수 없음

    예시

    변수범주순서
    학력초졸, 중졸, 고졸, 대졸, 대학원졸1 < 2 < 3 < 4 < 5
    통증 등급없음, 경미, 중등, 심각0 < 1 < 2 < 3
    만족도매우불만, 불만, 보통, 만족, 매우만족1 < 2 < 3 < 4 < 5
    암 병기Stage I, II, III, IVI < II < III < IV

    가능한 연산과 분석

    • 연산: =, ≠, <, >, ≤, ≥
    • 대표값: 중앙값(Median), 최빈값
    • 산포도: 백분위수, 사분위 범위(IQR)
    • 분석:
      • Mann-Whitney U 검정
      • Wilcoxon 검정
      • Kruskal-Wallis 검정
      • 스피어만 상관 (Spearman's ρ)

    DB 저장:

    -- 순서를 반영하는 정수로 저장
education_level INTEGER CHECK(education_level BETWEEN 1 AND 5)

-- 의미 있는 텍스트와 순서를 함께 저장
CREATE TABLE education_levels (
    code INTEGER PRIMARY KEY,
    label TEXT,
    display_order INTEGER
);

INSERT INTO education_levels VALUES
(1, '초등학교 졸업', 1),
(2, '중학교 졸업', 2),
(3, '고등학교 졸업', 3),
(4, '대학교 졸업', 4),
(5, '대학원 졸업', 5);

    3. 등간 척도 (Interval Scale)

    등간 척도는 범주 간 간격이 균등하지만, 절대 영점이 없습니다.

    특성

    • 간격 균등: 10°C와 20°C의 차이 = 20°C와 30°C의 차이
    • 절대 0 없음: 0이 "없음"을 의미하지 않음
    • 비율 무의미: 20°C가 10°C보다 "2배 덥다"고 할 수 없음
    • 덧셈/뺄셈 가능: 차이 계산에 의미 있음

    예시

    변수설명0의 의미
    섭씨 온도 (°C)물의 어는 점을 0으로임의의 기준점
    IQ 점수평균 100, SD 15로 표준화IQ 0은 없음
    연도 (AD)기원후 연도0년은 임의의 기준
    표준점수 (Z)평균 0, SD 1로 변환평균이 0일 뿐

    가능한 연산과 분석

    • 연산: =, ≠, <, >, +, −
    • 대표값: 평균(Mean), 중앙값, 최빈값
    • 산포도: 표준편차, 분산
    • 분석:
      • t-검정
      • 분산분석(ANOVA)
      • 피어슨 상관(Pearson's r)
      • 회귀분석

    4. 비율 척도 (Ratio Scale)

    비율 척도는 절대 영점이 있어서 비율 계산이 의미 있는 척도입니다.

    특성

    • 절대 0 있음: 0이 "완전히 없음"을 의미
    • 비율 의미 있음: 100kg은 50kg의 2배
    • 모든 수학 연산 가능: +, −, ×, ÷
    • 가장 높은 측정 수준

    예시

    변수단위0의 의미
    키cm0cm = 키 없음
    체중kg0kg = 질량 없음
    반응 시간ms0ms = 즉각 반응
    소득원0원 = 소득 없음
    거리m0m = 거리 없음
    켈빈 온도K0K = 절대 영도

    가능한 연산과 분석

    • 연산: =, ≠, <, >, +, −, ×, ÷
    • 대표값: 평균, 중앙값, 최빈값, 기하평균
    • 산포도: 표준편차, 분산, 변이계수(CV)
    • 분석: 모든 통계 분석 가능

    등간 vs 비율 척도 구분법

    질문: "0이 '없음'을 의미하는가?"

    • 예 → 비율 척도 (예: 체중 0kg = 질량 없음)
    • 아니오 → 등간 척도 (예: 0°C = 그냥 차가움)

    5. 척도별 가능한 연산과 분석

    종합 비교표

    특성 명목 서열 등간 비율
    분류 O O O O
    순서 X O O O
    균등 간격 X X O O
    절대 영점 X X X O
    가능한 연산 =, ≠ =, ≠, <, > =, ≠, <, >, +, − 모든 연산
    대표값 최빈값 중앙값 평균 평균, 기하평균
    대표 분석 χ² Mann-Whitney t-test, ANOVA 모든 분석

    상위 척도의 특권

    상위 척도는 하위 척도의 모든 분석이 가능합니다:

    • 비율 척도 → 등간, 서열, 명목 분석 모두 가능
    • 등간 척도 → 서열, 명목 분석 가능
    • 서열 척도 → 명목 분석 가능

    그러나 하위 척도에 상위 척도의 분석을 적용하면 안 됩니다 (예: 명목 척도에 평균 계산).

    6. 리커트 척도: 서열인가 등간인가?

    리커트 척도(Likert Scale)는 연구에서 가장 흔히 사용되는 척도입니다. 그러나 이것이 서열 척도인지 등간 척도인지에 대해 오랜 논쟁이 있습니다.

    리커트 척도란?

    동의 정도를 묻는 응답 형식입니다.

    12345
    전혀 아니다아니다보통이다그렇다매우 그렇다

    논쟁

    서열 척도로 보는 견해등간 척도로 보는 견해
    1→2와 4→5의 심리적 간격이 다를 수 있음 응답자는 숫자를 균등 간격으로 인식
    엄밀히 순서만 보장됨 실증 연구에서 등간으로 가정해도 결과 유사
    비모수 검정 사용해야 중심극한정리로 모수 검정 가능

    실무적 가이드라인

    • 단일 문항 (Likert Item):
      • 엄밀히는 서열 척도
      • 비모수 검정 권장 (중앙값, Mann-Whitney)
    • 합산 점수 (Likert Scale):
      • 여러 문항의 합/평균
      • 등간 척도로 처리 가능
      • 모수 검정 사용 가능 (평균, t-test)

    예시: 우울 척도 PHQ-9

    • 단일 문항: "기분이 가라앉거나 우울했다" → 0, 1, 2, 3 (서열)
    • 총점: 9개 문항 합산 → 0~27 (등간으로 처리)

    7. DB 데이터 타입 매핑

    척도 수준에 따라 적절한 DB 데이터 타입을 선택해야 합니다.

    척도 → DB 타입 대응표

    척도권장 DB 타입예시이유
    명목 TEXT 또는 INTEGER + FK gender TEXT, blood_type_id INTEGER 범주 표현, 코드 테이블 연결
    서열 INTEGER education_level INTEGER 순서 보존, 비교 연산 가능
    등간 REAL 또는 DECIMAL iq_score REAL 소수점 가능, 연산 지원
    비율 REAL 또는 DECIMAL weight_kg REAL, height_cm REAL 모든 연산 지원

    종합 예시:

    CREATE TABLE participants (
    id INTEGER PRIMARY KEY,

    -- 명목 척도
    gender TEXT CHECK(gender IN ('M', 'F')),
    diagnosis_code TEXT,

    -- 서열 척도
    education_level INTEGER CHECK(education_level BETWEEN 1 AND 5),
    pain_grade INTEGER CHECK(pain_grade BETWEEN 0 AND 3),

    -- 등간 척도 (또는 등간으로 처리)
    depression_phq9 INTEGER,  -- 0-27, 합산 점수
    iq_score REAL,

    -- 비율 척도
    age INTEGER,              -- 절대 0 있음
    height_cm REAL,
    weight_kg REAL,
    reaction_time_ms REAL
);

-- 변수 메타데이터에 척도 수준 기록
CREATE TABLE variable_metadata (
    variable_name TEXT PRIMARY KEY,
    scale_level TEXT CHECK(scale_level IN
        ('nominal', 'ordinal', 'interval', 'ratio')),
    description TEXT
);

INSERT INTO variable_metadata VALUES
('gender', 'nominal', '성별'),
('education_level', 'ordinal', '학력 수준'),
('depression_phq9', 'interval', 'PHQ-9 우울 점수'),
('weight_kg', 'ratio', '체중 (kg)');
