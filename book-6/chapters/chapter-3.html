📘 6권: 연구 데이터 설계

변수부터 회귀분석까지

    1부: 연구와 데이터의 기초

    [3장] 변수란 무엇인가

    이 장에서는 변수의 정의와 측정에 대해 배웁니다. 추상적인 개념을 어떻게 측정 가능한 변수로 만드는지가 연구의 핵심입니다.

    1. 변수의 정의

    변수(Variable)는 관측 대상에 따라 다른 값을 가지는 특성입니다.

    변수와 상수

    • 변수(Variable): 관측 단위마다 다른 값을 가짐
      예: 나이, 키, 점수, 성별
    • 상수(Constant): 모든 관측 단위에서 같은 값
      예: 원주율(π), 연구 수행 연도

    연구에서는 변수 간의 관계를 분석합니다. 상수는 분석 대상이 아닙니다.

    변수의 값

    변수가 취할 수 있는 구체적인 수치나 범주입니다.

    변수가능한 값
    성별남, 여
    학력초졸, 중졸, 고졸, 대졸, 대학원졸
    통증 강도 (VAS)0~100의 정수
    체중 (kg)양의 실수

    DB에서 변수의 표현

    데이터베이스에서 변수는 열(column)로 표현됩니다. 각 행(row)은 하나의 관측 단위(참가자)를 나타냅니다.

    idagegenderpain_score
    145M65
    232F42
    358M78

    age, gender, pain_score는 변수(열)이고, 각 행의 값은 해당 참가자의 측정값입니다.

    2. 개념적 정의 vs 조작적 정의

    연구에서는 추상적 개념을 측정 가능한 형태로 변환해야 합니다. 이 과정에서 두 종류의 정의가 필요합니다.

    개념적 정의 (Conceptual Definition)

    사전적, 이론적 의미에서의 정의입니다. 추상적이고 일반적입니다.

    • 우울: 지속적인 슬픔, 흥미 상실, 무력감을 특징으로 하는 정서 상태
    • 삶의 질: 신체적, 정신적, 사회적 안녕의 주관적 인식
    • 통증: 실제 또는 잠재적 조직 손상과 관련된 불쾌한 감각 및 정서적 경험

    조작적 정의 (Operational Definition)

    개념을 어떻게 측정할 것인가를 구체적으로 명시한 정의입니다.

    • 우울: PHQ-9 점수 (0-27점), 10점 이상을 우울로 분류
    • 삶의 질: SF-36 설문의 8개 하위 척도 점수
    • 통증: 100mm VAS에서 참가자가 표시한 지점의 거리 (mm)

    예시: "운동"의 조작적 정의

    개념적 정의: 건강 증진을 위한 신체 활동

    조작적 정의 예시:

    1. "지난 일주일간 중강도 이상 운동을 30분 이상 실시한 일수" (0-7)
    2. "IPAQ 설문으로 측정한 주당 MET-minutes"
    3. "만보기로 측정한 일일 평균 걸음 수"
    4. "최대 산소섭취량(VO2max) 측정값 (mL/kg/min)"

    같은 개념도 조작적 정의에 따라 다른 값이 나올 수 있습니다.

    DB 설계: 변수의 정의 저장

    CREATE TABLE variable_definitions (
    id INTEGER PRIMARY KEY,
    variable_name TEXT NOT NULL,
    conceptual_definition TEXT,
    operational_definition TEXT,
    measurement_instrument TEXT,
    unit TEXT,
    min_value REAL,
    max_value REAL
);

    3. 측정과 측정 오차

    측정(Measurement)은 규칙에 따라 대상에 숫자나 기호를 부여하는 과정입니다.

    측정의 기본 공식

    관측값 = 참값 + 오차

    Xobserved = Xtrue + Error

    • 참값(True Score): 오차가 없을 때의 실제 값
    • 관측값(Observed Score): 실제로 측정된 값
    • 오차(Error): 참값과 관측값의 차이

    체계적 오차 (Systematic Error)

    일관된 방향으로 발생하는 오차입니다. 측정값이 한쪽으로 치우칩니다.

    • 원인:
      • 보정되지 않은 측정 도구
      • 일관된 측정자 편향
      • 환경 요인
    • 영향: 타당도를 낮춤 (엉뚱한 것을 측정)
    • 예: 영점이 맞지 않는 체중계 → 모든 측정값이 실제보다 2kg 높음

    무선 오차 (Random Error)

    무작위로 발생하는 오차입니다. 방향이 일정하지 않습니다.

    • 원인:
      • 피험자의 일시적 상태 (피로, 주의력)
      • 환경의 무작위 변동
      • 측정자의 비일관성
    • 영향: 신뢰도를 낮춤 (측정이 일관되지 않음)
    • 특성: 표본 크기가 커지면 평균적으로 상쇄됨

    체계적 오차 vs 무선 오차

    구분체계적 오차무선 오차
    방향일관됨 (한쪽으로)무작위 (양쪽으로)
    영향타당도 ↓신뢰도 ↓
    표본 증가 시상쇄 안 됨평균적으로 상쇄
    통제 방법도구 보정, 맹검측정 횟수 증가, 표준화

    4. 신뢰도와 타당도 개요

    측정의 질을 평가하는 두 가지 핵심 기준입니다.

    신뢰도 (Reliability)

    측정이 일관되게 반복되는 정도입니다.

    • 같은 대상을 반복 측정했을 때 같은 결과가 나오는가?
    • 무선 오차가 적을수록 신뢰도가 높음
    • 0에서 1 사이의 계수로 표현 (높을수록 좋음)

    신뢰도 유형:

    • 검사-재검사 신뢰도 (Test-retest)
    • 내적 일관성 (Internal consistency, Cronbach's α)
    • 평가자 간 신뢰도 (Inter-rater reliability)

    타당도 (Validity)

    측정이 의도한 것을 측정하는 정도입니다.

    • 측정하려는 개념을 실제로 측정하고 있는가?
    • 체계적 오차가 적을수록 타당도가 높음

    타당도 유형:

    • 내용 타당도: 측정 내용이 개념을 대표하는가
    • 준거 타당도: 외부 기준과 일치하는가 (동시/예측)
    • 구성 타당도: 이론적 구성개념을 측정하는가

    과녁 비유

    신뢰도 높음신뢰도 낮음
    타당도 높음 과녁 중심에 모여 있음
    (이상적)    		 과녁 중심 주변에 흩어져 있음
    (불가능: 신뢰도는 타당도의 필요조건)
    타당도 낮음 과녁 한쪽에 모여 있음
    (일관되게 빗나감)    		 사방에 흩어져 있음
    (최악)

    핵심 원리

    신뢰도는 타당도의 필요조건이지만 충분조건은 아닙니다.

    • 신뢰도가 낮으면 타당도도 낮습니다
    • 그러나 신뢰도가 높다고 타당도가 보장되지 않습니다
    • 수학적으로: 타당도 ≤ √신뢰도

    5. 변수명 짓기 규칙

    데이터베이스와 분석 도구에서 사용할 변수명은 규칙을 따라야 합니다.

    좋은 변수명의 조건

    • 의미 있음: 변수의 내용을 유추할 수 있음
    • 간결함: 불필요하게 길지 않음
    • 일관성: 같은 규칙을 모든 변수에 적용
    • 기계 친화적: 특수문자, 공백 없음

    명명 규칙 (Naming Convention)

    규칙예시사용처
    snake_case pain_score, birth_year SQL, Python, R
    camelCase painScore, birthYear JavaScript, Java
    PascalCase PainScore, BirthYear C#, 테이블명
    SCREAMING_SNAKE PAIN_SCORE 상수, 일부 SQL

    권장: 연구 데이터에서는 snake_case가 가독성이 좋고 SQL, R, Python 모두에서 잘 작동합니다.

    변수명 작성 팁

    • 접두사 활용:
      • pre_, post_: 사전/사후 측정
      • t1_, t2_: 시점
      • is_, has_: 이진 변수
    • 접미사 활용:
      • _score: 점수
      • _date: 날짜
      • _id: 식별자
      • _cat: 범주형

    피해야 할 변수명

    • 공백 포함: pain score → pain_score
    • 특수문자: pain-score, pain@score
    • 숫자로 시작: 1st_score → score_1st
    • 예약어: select, from, where, date
    • 너무 짧음: x, v1, a
    • 너무 김: patient_pain_intensity_visual_analog_scale_score_at_baseline
    • 한글: 통증점수 (일부 도구에서 오류)

    예시: 연구 변수명 설계

    -- 좋은 변수명 예시
CREATE TABLE assessments (
    id INTEGER PRIMARY KEY,
    participant_id INTEGER,

    -- 인구통계
    age INTEGER,
    gender TEXT,
    education_years INTEGER,

    -- 사전 측정
    pre_pain_vas REAL,
    pre_function_odi REAL,
    pre_depression_phq9 INTEGER,

    -- 사후 측정
    post_pain_vas REAL,
    post_function_odi REAL,
    post_depression_phq9 INTEGER,

    -- 메타 정보
    is_completer BOOLEAN,
    assessment_date DATE
);
