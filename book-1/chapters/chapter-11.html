데이터베이스 기초 개념

    [부록] 데이터베이스 영어 용어집

    개발자와 소통하거나 영어 문서 읽을 때 필요한 핵심 용어들!

    기본 구조 용어

    영어한글설명
    Database (DB)데이터베이스데이터 저장소 전체
    Table테이블데이터를 담는 표
    Row / Record행 / 레코드데이터 한 건
    Column / Field열 / 필드 / 컬럼데이터 항목
    Schema스키마DB 전체 구조 설계도
    Entity엔티티테이블로 표현되는 대상 (환자, 치료사 등)
    Attribute속성엔티티의 특성 (이름, 연락처 등)

    키 (Key) 관련

    영어한글설명
    Primary Key (PK)기본키레코드를 유일하게 식별하는 키
    Foreign Key (FK)외래키다른 테이블의 PK를 참조
    Candidate Key후보키PK가 될 수 있는 키들
    Composite Key복합키2개 이상 컬럼으로 만든 키
    Unique Key유니크키중복 불가 (NULL은 가능)
    Index인덱스검색 속도 향상용 색인

    정규화 (Normalization)

    영어한글설명
    Normalization정규화테이블을 체계적으로 분리하는 과정
    Denormalization비정규화/역정규화성능을 위해 정규화를 푸는 것
    1NF (First Normal Form)제1정규형원자값만 저장
    2NF (Second Normal Form)제2정규형부분 종속 제거
    3NF (Third Normal Form)제3정규형이행 종속 제거
    BCNF (Boyce-Codd NF)보이스-코드 정규형결정자가 모두 후보키
    Anomaly이상 현상삽입/갱신/삭제 이상
    Redundancy중복같은 데이터가 여러 곳에

    종속성 (Dependency)

    영어한글설명
    Functional Dependency함수 종속A → B (A가 B를 결정)
    Partial Dependency부분 종속복합키 일부에만 종속
    Transitive Dependency이행 종속A → B → C 연쇄 종속
    Multi-valued Dependency다치 종속독립적인 여러 값 집합
    Determinant결정자다른 값을 결정하는 속성

    관계 (Relationship)

    영어한글설명
    One-to-One (1:1)일대일1개 : 1개
    One-to-Many (1:N)일대다1개 : 여러 개
    Many-to-Many (N:M)다대다여러 개 : 여러 개
    Junction Table연결 테이블 / 중간 테이블N:M 관계를 풀어주는 테이블
    ERD (Entity Relationship Diagram)개체 관계도테이블 관계를 그린 다이어그램
    Cardinality카디널리티관계의 수적 표현 (1:N 등)

    SQL 명령어

    영어한글용도
    SELECT선택/조회데이터 가져오기
    INSERT삽입데이터 추가
    UPDATE갱신/수정데이터 변경
    DELETE삭제데이터 제거
    CREATE생성테이블/DB 만들기
    DROP삭제테이블/DB 제거
    ALTER변경테이블 구조 수정
    JOIN조인/결합테이블 연결 조회
    WHERE조건절필터링 조건
    GROUP BY그룹화집계 기준
    ORDER BY정렬결과 순서 지정
    HAVING그룹 조건GROUP BY 결과 필터링

    JOIN 종류

    영어한글설명
    INNER JOIN내부 조인양쪽 다 있는 것만
    LEFT JOIN왼쪽 조인왼쪽 테이블 기준 전체
    RIGHT JOIN오른쪽 조인오른쪽 테이블 기준 전체
    FULL OUTER JOIN완전 외부 조인양쪽 테이블 전체
    CROSS JOIN교차 조인모든 조합 (곱집합)
    SELF JOIN셀프 조인같은 테이블끼리 조인

    데이터 타입

    영어한글예시
    VARCHAR가변 문자열이름, 주소
    CHAR고정 문자열성별 (M/F)
    INT / INTEGER정수나이, 금액
    DECIMAL / NUMERIC소수ROM 각도
    DATE날짜2024-01-15
    TIME시간14:30:00
    DATETIME / TIMESTAMP날짜+시간등록일시
    BOOLEAN / BOOL참/거짓TRUE/FALSE
    TEXT긴 문자열SOAP 기록
    BLOB바이너리이미지, 파일
    NULL널/없음값이 없는 상태

    제약조건 (Constraints)

    영어한글설명
    NOT NULL필수값NULL 불가
    UNIQUE유일중복 불가
    DEFAULT기본값값 없으면 자동 지정
    CHECK체크값 범위 제한
    AUTO_INCREMENT자동 증가1, 2, 3... 자동 부여
    REFERENCES참조외래키 지정

    💡 개발자와 대화할 때

    "이 테이블의 PK가 뭐예요?"
    → 기본키가 뭔지 물어보는 것

    "FK로 연결해주세요"
    → 외래키로 다른 테이블 참조하게 해달라는 것

    "N:M 관계라서 Junction Table이 필요해요"
    → 다대다라서 중간 테이블 만들어야 한다는 것

    "3NF까지 정규화 해주세요"
    → 제3정규형까지 테이블 분리해달라는 것

    "이 컬럼 NOT NULL로 해주세요"
    → 필수 입력값으로 만들어달라는 것
